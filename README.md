# Pokemon Analyzer

포켓몬 이미지를 입력하면 가장 유사한 포켓몬을 분석해 Top-5 결과를 보여주는 Python GUI 프로그램입니다.

HTML이나 웹 서버 없이 Python의 `tkinter`를 이용해 실행되며, 이미지 삽입 기능과 간단한 그림판 기능을 제공합니다.

---

## 주요 기능

- 포켓몬 이미지 삽입 기능
- 검은색 펜을 이용한 간단한 그림판 입력 기능
- 분석 중 몬스터볼 로딩 화면 및 진행률 표시
- Top-5 예측 결과 출력
- `data/PokemonData` 폴더 기반의 포켓몬 이미지 분석
- 모델 파일이 없을 경우 사용자가 직접 모델 생성 가능

## 데이터셋 준비

이 프로그램은 포켓몬 이미지 데이터셋을 사용합니다.

Kaggle 데이터셋을 다운로드한 뒤, `PokemonData` 폴더를 아래 위치에 넣습니다.
Kaggle 데이터셋 다운로드: [Kaggle Pokémon Dataset](https://www.kaggle.com/datasets/lantian773030/pokemonclassification?resource=download)

```text
data/PokemonData/
```

폴더 구조는 다음과 같아야 합니다.

```text
data/
└─ PokemonData/
   ├─ Abra/
   │  ├─ image1.jpg
   │  └─ image2.jpg
   ├─ Pikachu/
   │  ├─ image1.jpg
   │  └─ image2.jpg
   └─ ...
```

주의할 점은 `data` 폴더 안에 이미지 파일만 직접 넣으면 안 된다는 것입니다.
반드시 포켓몬 이름별 폴더가 있어야 합니다.

## 실행 방법

### 방법 1. bat 파일로 실행

Windows 환경에서는 아래 파일을 더블클릭합니다.

```text
start_app.bat
```

### 방법 2. Python 파일 직접 실행

터미널에서 다음 명령어를 실행합니다.

```bash
python pokemon_gui.py
```

## 사용 방법

1. 프로그램을 실행합니다.
2. 처음 실행하는 경우 `[모델 만들기]` 버튼을 누릅니다.
3. 모델 생성이 완료되면 이미지를 삽입하거나 그림판에 포켓몬을 그립니다.
4. `[분석]` 버튼을 누릅니다.
5. 오른쪽 화면에서 로딩 후 Top-5 분석 결과를 확인합니다.

## 실행 결과 및 고찰

이미지 삽입 기능을 이용한 분석에서는 일반적인 포켓몬 이미지에 대해 비교적 잘 인식하는 결과를 확인할 수 있었습니다. 특히 데이터셋에 포함된 이미지와 비슷한 구도, 색상, 형태를 가진 경우에는 상위 예측 결과에 올바른 포켓몬이 포함되는 경우가 많았습니다.

다만 포켓몬이 특이한 포즈를 취하고 있는 경우에는 인식 성능이 떨어지는 문제가 있었습니다. 예를 들어 피카츄가 일반적인 정면 또는 측면 자세가 아니라, 고라파덕처럼 머리를 잡고 있는 특이한 자세를 취할 경우 이미지의 형태적 특징이 달라져 예측이 저해되었습니다. 이는 현재 모델이 이미지의 색상과 형태적 유사도에 크게 의존하기 때문으로 보입니다.

그림판 기능도 추가하였지만, 실제 사진 이미지와 사용자가 직접 그린 그림 사이에는 차이가 크기 때문에 분석 정확도는 아직 개선이 필요합니다. 현재 그림판은 검은색 선만 지원하므로 색상 정보가 부족하고, 사용자의 그림 실력이나 선의 형태에 따라 결과가 크게 달라질 수 있습니다. 향후에는 색상 선택, 브러시 굵기 조절, 지우개 기능, 그림 데이터 기반 추가 학습 등이 필요할 것으로 보입니다.

또한 더 다양한 이미지와 더 큰 데이터셋을 사용하면 모델의 일반화 성능이 향상될 것으로 예상됩니다. 특히 다양한 포즈, 각도, 배경, 그림체를 포함한 데이터가 추가된다면 특이한 이미지에 대한 인식 성능도 개선될 수 있습니다.

흥미로운 점은 여러 분석 결과에서 Mew가 체감상 자주 등장했다는 것입니다. Mew는 포켓몬 세계관에서 모든 포켓몬의 유전자를 가진 포켓몬이라는 설정을 가지고 있는데, 다양한 포켓몬 이미지와 특징적으로 유사하게 나타나는 결과가 이 설정과 어느 정도 부합하는 것처럼 느껴졌습니다. 이 현상이 단순히 모델의 특징 추출 방식 때문인지, 데이터셋의 이미지 분포 때문인지에 대해서는 후속 분석이 필요합니다.

## 한계점

현재 프로그램은 TensorFlow 기반의 딥러닝 전이학습 모델이 아니라, 색상과 형태 특징을 이용한 간단한 이미지 유사도 기반 classifier입니다.

따라서 다음과 같은 한계가 있습니다.

* 특이한 포즈의 포켓몬 이미지에 약함
* 배경이 복잡한 이미지에 약함
* 그림판으로 그린 단순한 그림의 인식률이 낮음
* 색상이 비슷한 포켓몬끼리 혼동할 수 있음
* 실제 전이학습 기반 CNN 모델보다 성능이 낮을 수 있음

---

## 개선 방향

향후 개선 방향은 다음과 같습니다.

1. MobileNetV2, ResNet, EfficientNet 등의 backbone 모델을 사용한 전이학습 적용
2. 포켓몬 포즈와 각도 다양화를 위한 데이터 증강 적용
3. 그림판 입력용 별도 데이터셋 구축
4. 색상 선택, 브러시 크기 조절, 지우개 기능 추가
5. 테스트 정확도, precision, recall 등 정량적 성능 평가 추가
6. 더 많은 포켓몬 이미지 데이터 확보
7. Top-5 결과뿐 아니라 예측 근거를 시각적으로 보여주는 기능 추가
